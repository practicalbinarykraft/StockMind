Окей, вижу: тост «Статья переведена», но модалка закрывается, а при повторном открытии — снова английский. Это почти наверняка смесь из двух багов: (а) кнопка запуска перевода закрывает модалку (submit/propagation), (б) перевод не сохраняется/затирается автоподгрузкой оригинала.

Вот что нужно поправить, по пунктам — быстро и без ломки текущего флоу.

Что починить
	1.	Кнопка «Перевести» не должна закрывать модалку

	•	Убедись, что это <button type="button">, а не type="submit".
	•	Если она внутри формы/диалога — добавь onClick={(e) => { e.preventDefault(); e.stopPropagation(); … }}.
Иначе клик уходит наверх и триггерит close.

	2.	Сброс/хранение состояния перевода по articleId

	•	В модалке держи состояние вида:

const [translated, setTranslated] = useState<{title: string; content: string} | null>(null)
const [isTranslated, setIsTranslated] = useState(false)


	•	При смене статьи сбрасывай перевод:

useEffect(() => { setTranslated(null); setIsTranslated(false); }, [articleId])


	•	Не закрывай модалку на успешный перевод: просто выставляй setTranslated({title, content}); setIsTranslated(true), меняй кнопку на «Показать оригинал».

	3.	Не затирать перевод автоподгрузкой «полного текста»
Сейчас после GET /news/:id/full у тебя эффект типа:

useEffect(() => {
  if (fullText) setContent(fullText) // ← затирает перевод
}, [fullText])

Добавь охрану:

useEffect(() => {
  if (!isTranslated && fullText) setContent(fullText)
}, [isTranslated, fullText])

Или вообще рендери: isTranslated ? translated.content : fullText.
	4.	Нормально кэшировать перевод (чтобы при повторном открытии был RU)

	•	Введи React Query-ключ ['article-translation', articleId, 'ru'].
	•	После успешного перевода:

queryClient.setQueryData(['article-translation', articleId, 'ru'], { title, content })
setTranslated({ title, content })
setIsTranslated(true)


	•	При открытии модалки сначала пробуй useQuery этого ключа: если есть — показывай сразу перевод (кнопку — «Показать оригинал»).

	5.	Серверный ответ и размеры
Проверь, что POST /api/news/:id/translate возвращает ровно:

{ "title": "…ru", "content": "…ru", "lang": "ru" }

и фронт именно эти поля мапит. Если текст длинный — уже есть у нас чанкинг и safe-parse; верни склеенный content. Не возвращай «success: true» без текста.
	6.	UI-мелочи, без которых UX ломается

	•	Кнопка «Перевести» залочена, пока fullText загружается; после перевода — меняется на «Показать оригинал»/«Показать перевод».
	•	Не закрывать модалку ни при успехе, ни при ошибке. Только тост + инлайн-стейт.
	•	Если paywall/парсер не отдал полный текст — покажи в модалке предупреждение и всё равно разреши переводить body/summary (если есть).

Мини-чеклист для проверки
	1.	Открыть «Предпросмотр» → дождаться «полного текста» (скелетон исчез).
	2.	Нажать «Перевести» → модалка остаётся открытой, текст становится RU, кнопка меняется на «Показать оригинал».
	3.	Закрыть модалку, открыть снова ту же карточку → сразу вижу RU (из кэша).
	4.	Переключить на другую статью → перевод сброшен (EN), кнопка снова «Перевести».

Почему это происходило
	•	Кнопка имела поведение submit/propagation → диалог закрывался.
	•	Эффект загрузки «полного текста» перезаписывал уже переведённый контент.
	•	Перевод не кэшировался по articleId, поэтому при повторном открытии модалка не знала про RU-результат.

Сделай эти 5 правок — и перевод в предпросмотре будет стабильным и предсказуемым.
